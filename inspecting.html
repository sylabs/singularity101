ðŸ“„ Inspecting SIF Files#

Singularity stores metadata in their SIF files. For example, if you want to see the original definition file you can use the inspect command:

$ singularity inspect --deffile 

bootstrap: library
from: ubuntu:22.04

You can also use the following prompts to show the following sections:

  • -r to show the runscript section for the image.

  • -s to show the startscript section.

  • -t to show the test script.

  • -e to show the environment settings.

Accessing host files#

In a container, certain directories such as $HOME, /tmp, /proc, /sys and /dev are usually accessible. While it is possible to access, create, and modify files and directories on the host system from within the container, the hostâ€™s user permissions will always be honored.

To make other directories accessible, you can use the â€“bind option. For example, if you want to access a specific directory or file, you can make it available inside the container.

$ singularity exec --bind /etc/os-release library://ubuntu:22.04 cat /etc/os-release

Note how the first command shows Ubuntu information and the second command shows the information about AlmaLinux, which is the file stored in the host.

To mount multiple directories, you can provide them as a comma-separated list like the following:

$ mkdir -p {run,var-log}
$ singularity exec --bind run:/run,var-log:/var/log library://ubuntu:22.04 ls -l /var/log

The previous command lists the files inside the hostâ€™s /var/log directory.
